MNIST linear-query classifier

A Hebbian-style classifier, output perturbation, effective dimension, and the return to reconstruction.

1. Present the MNIST data set

MNIST is a collection of handwritten digit images. Each example is an image vector \(x_i\in\mathbb R^{784}\) with a digit label. For privacy, the private row is the labeled example \((x_i,y_i)\).

The previous lecture ended with private linear regression: the model was post-processing of noisy sufficient statistics. Here the first model will be even simpler: the classifier itself is one vector query.

The images themselves should be read as grayscale pixel intensities. The plots later in the lecture use color when they show signed weights, accuracy curves, or noise scales.

3. Propose a Hebbian-style linear classifier

Use the signed sum

\[ S=\sum_i y_i x_i, \]

or equivalently the signed average

\[ w=\frac{1}{n}\sum_i y_i x_i. \]

The scale does not change the decision boundary: predictions use \(\operatorname{sign}(w^\top x)\). This is the classic Hebbian intuition: pixels that repeatedly co-occur with the positive label get positive weight, and pixels that co-occur with the negative label get negative weight.

4. What it means to be a linear classifier

In two dimensions, a linear classifier is a line. The vector \(w\) points toward the side predicted as positive; the decision boundary is the set of points with \(w^\top x=0\).

This plot is an explicit two-dimensional teaching example. It is not MNIST PCA. Its only job is to show what the phrase “linear classifier” means before we return to the 784-pixel MNIST query.

5. Two means and the classifier image

For 0 vs 1, the signed average is a difference of sums:

\[ w=\frac{1}{n}\sum_i y_i x_i = \frac{1}{n}\left(\sum_{y_i=+1}x_i-\sum_{y_i=-1}x_i\right). \]

Equivalently,

\[ w=\frac{n_+}{n}\bar x_+ - \frac{n_-}{n}\bar x_-. \]

When the two classes are nearly balanced, this is proportional to mean 1 minus mean 0. The image below shows the two grayscale class means and the colored signed classifier weights.

6. This is a linear query, so use output perturbation

Because \(w\) is an average of bounded rows, replacing one labeled example changes it by at most

\[ \Delta_2(w)\le \frac{2}{n}. \]

So the natural mechanism is output perturbation:

\[ \widetilde w=w+\mathcal N(0,\sigma^2 I), \]

with \(\sigma\) calibrated to \(2/n\). Unlike Lecture 7, there is no noisy matrix inverse here; the released object is the query output itself.

13. Effective dimension: why noise does not win immediately

The intuitive answer is that MNIST is not 784 unrelated directions. Most of the useful signal lies in a smaller subspace, and different public tasks use that subspace differently. The spectrum plot shows how quickly signed-image energy accumulates; the accuracy plot asks how much test accuracy remains when the classifier is projected onto the leading directions.

15. Membership bridge: a real row contributes pixels to the query

The classifier is

\[ w=\frac{1}{n}\sum_i y_i x_i. \]

So every real training row contributes \(y_i x_i/n\) to the released vector. The plot chooses a real low-margin training row from the actual training set. It shows the row, the signed term \(y_i x_i\), and the specific pixels that this row adds to \(w\). Membership inference asks whether a released model retains enough row-specific signal for an attacker to distinguish training rows from non-training rows.

make_training_row_contribution_figure(artifact)
